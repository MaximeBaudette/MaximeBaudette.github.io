## Build Commands

```bash
# First-time setup (after clone or npm dep changes)
npm ci
npm run setup  # copies assets from node_modules + downloads CSS/JS

# Local dev server (requires Homebrew Ruby 3.4, not system Ruby)
PATH="/opt/homebrew/opt/ruby@3.4/bin:/usr/local/bin:/usr/bin:/bin:$PATH" \
  bundle exec jekyll serve --livereload --port 4000

# Production build
JEKYLL_ENV=production RUBYOPT=-EUTF-8 bundle exec jekyll build

# Install Ruby dependencies
PATH="/opt/homebrew/opt/ruby@3.4/bin:/usr/local/bin:/usr/bin:/bin:$PATH" \
  bundle install
```

No tests exist -- build success is the validation.

## Architecture

Jekyll + al-folio theme. Liquid templates, SCSS, Jekyll plugins.

### Build Pipeline
1. `npm run setup` copies assets from `node_modules/` to `assets/` + `_sass/`
2. `bundle exec jekyll build` runs plugins, compiles SCSS, renders Liquid, outputs to `_site/`
3. **PurgeCSS** strips unused CSS from `_site/assets/css/*.css` (runs after Jekyll, before deploy)
4. Cloudflare Pages deploys `_site/` automatically on push to `master`

## Build Pipeline Changes

### 1. PurgeCSS Integration
**File:** `purgecss.config.js`

Runs after Jekyll build to strip unused CSS classes:
- Content: `_site/**/*.html`, `_site/**/*.js`
- CSS input: `_site/assets/css/*.css`
- CSS output: `_site/assets/css/` (overwrites)
- Safelist: theme classes (`light-*`, `dark-*`), Bootstrap dynamic classes (`show`, `active`, `collapsing`, etc.)

**Build command:**
```
npm ci && npm run setup && bundle install && JEKYLL_ENV=production RUBYOPT=-EUTF-8 bundle exec jekyll build && npx purgecss --config purgecss.config.js
```

### 2. Tabler Icons Removal
**Rationale:** The homepage only uses 0 Tabler icons. The full set was ~8MB of font files + 5000+ CSS rules.

**Changes:**
- Removed `_sass/tabler-icons-minimal.scss` (was our custom minimal subset)
- Removed `tabler-icons-minimal` import from `assets/css/main.scss`
- Removed all Tabler font files from `assets/fonts/`
- Removed Tabler copy lines from `scripts/setup-assets.sh`
- Removed `$ti-font-path` variable from `assets/css/main.scss`

**Result:** Saved ~8MB in deployed assets, ~5000 CSS rules.

### 3. FontAwesome Purge
**Rationale:** Only ~20 FA icons are used across the site (social icons, theme toggle, search).

**Changes:**
- Removed `fa-*` and `ti-*` from PurgeCSS safelist (they were there unnecessarily)
- PurgeCSS now strips unused FA icon classes from `_site/assets/css/main.css`

**Result:** main.css reduced from 131KB to 24KB (81% reduction).

### 4. Cache-Bust Plugin Fix
**File:** `_plugins/cache-bust.rb`

**Bug:** `bust_css_cache` used directory `assets/_sass` which doesn't exist. This caused the MD5 hash to always be `d41d8cd98f00b204e9800998ecf8427e` (empty string), defeating cache busting.

**Fix:** Changed to `_sass` (correct directory).

**Impact:** CSS URLs now have unique hashes when SCSS changes, ensuring browsers fetch fresh CSS.

### 5. Resource Hints
**File:** `_includes/head.liquid`

Added to `<head>`:
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link rel="preconnect" href="https://cdn.jsdelivr.net">
<link rel="dns-prefetch" href="https://fonts.googleapis.com">
<link rel="dns-prefetch" href="https://fonts.gstatic.com">
<link rel="dns-prefetch" href="https://cdn.jsdelivr.net">
```

Also added image preload for LCP element:
```html
<link rel="preload" as="image" href="{{ '/assets/img/profile_picture_homepage.webp' | relative_url }}">
```

### 6. Encoding Fix
**File:** `_config.yml` added `encoding: UTF-8`, build command adds `RUBYOPT=-EUTF-8`.

**Rationale:** Cloudflare Pages build machines have non-UTF-8 locale, causing Jekyll to fail on special characters.
